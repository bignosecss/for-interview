防抖的核心逻辑：
1. 接收一个方法，和延时
2. 触发防抖之后，如果在延时内再次触发防抖，那么取消之前的防抖
3. 只有在整个延时过程中都没有再次调用，才最终开始执行防抖函数

```javascript
function myDebounce(fn, delay) {
  let timer = null;

  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  }
}
```

节流的核心逻辑：
1. 接收一个方法，和延时
2. 第一次触发节流，马上执行方法，忽略延时内触发的节流

```javascript
function myThrottle(fn, delay) {
  let shouldRun = true;

  return function (...args) {
    if (!shouldRun) return;

    shouldRun = false;
    fn.apply(this, args);
    setTimeout(() => { shouldRun = true }, delay);
  }
}
```
